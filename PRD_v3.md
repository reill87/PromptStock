# PRD: 포트폴리오 분석 트래커

## 1. 제품 개요

### 1.1 제품명
Portfolio Analysis Tracker (가칭)

### 1.2 제품 설명
포트폴리오 분석 결과를 체계적으로 기록하고, 시간에 따른 변화를 추적할 수 있는 개인 투자 기록 도구. 사용자는 검증된 프롬프트 템플릿으로 LLM 분석을 수행하고, 그 결과를 앱에 저장하여 과거 분석과 비교할 수 있음.

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| 분석 히스토리 | LLM은 과거를 기억 못함. 이 앱은 모든 분석을 기록 |
| 시계열 비교 | 3개월 전 vs 지금, 포트폴리오가 어떻게 변했는지 추적 |
| 검증된 템플릿 | 매번 프롬프트 고민 없이 바로 분석 시작 |
| 비용 제로 | 사용자가 자신의 무료 LLM 계정 활용 |

### 1.4 타겟 유저

- 정기적으로 포트폴리오를 점검하는 개인 투자자
- 투자 의사결정 과정을 기록으로 남기고 싶은 사람
- LLM을 활용한 투자 분석에 관심 있는 사람

### 1.5 핵심 차별화

> **"ChatGPT/Claude는 당신의 3개월 전 포트폴리오를 기억하지 못합니다. 우리는 기억합니다."**

## 2. 사용자 플로우

### 2.1 메인 플로우

```
1. 템플릿 선택
   ↓
2. 프롬프트 복사
   ↓
3. 사용자가 직접 ChatGPT/Claude에서 분석
   (이미지 업로드 + 프롬프트 붙여넣기)
   ↓
4. 분석 결과를 앱에 저장
   (스크린샷 + LLM 응답 텍스트)
   ↓
5. 시간이 지나면 과거 분석과 비교
```

### 2.2 비교 분석 플로우

```
1. 히스토리에서 과거 분석 선택
   ↓
2. "비교하기" 버튼 클릭
   ↓
3. 현재 분석과 나란히 비교
   ↓
4. 변화 포인트 확인 (종목 변경, 비중 변화 등)
```

## 5. 데이터 모델

### 5.1 Template

```typescript
interface Template {
  id: string;
  name: string;
  category: 'risk' | 'rebalance' | 'checklist' | 'sector' | 'profit';
  description: string;
  icon: string;
  promptText: string;
  usageGuide: string;  // 사용법 설명
  exampleResult?: string;
  isCustom: boolean;
  createdAt: string;
  usageCount: number;
}
```

### 5.2 AnalysisRecord

```typescript
interface AnalysisRecord {
  id: string;
  createdAt: string;
  updatedAt: string;

  // 분석 내용
  templateId: string;
  templateName: string;
  portfolioImages: string[];  // FileSystem 경로
  llmResponse: string;
  usedPrompt?: string;  // 실제 사용한 프롬프트 (수정했을 경우)

  // 사용자 입력
  userNote?: string;
  tags: string[];

  // 포트폴리오 스냅샷 (선택적)
  snapshot?: PortfolioSnapshot;

  // 비교용 메타데이터
  comparedWith?: string[];  // 비교한 다른 분석 ID
}

interface PortfolioSnapshot {
  totalValue?: number;
  stockCount?: number;
  topHoldings?: StockHolding[];
}

interface StockHolding {
  name: string;
  ratio?: number;  // 비중 %
  value?: number;  // 평가금액
}
```

### 5.3 UserSettings

```typescript
interface UserSettings {
  // 기본 설정
  defaultTemplate?: string;
  autoSaveEnabled: boolean;
  imageQuality: 'low' | 'medium' | 'high';

  // 커스텀 템플릿
  customTemplates: Template[];

  // 알림 설정 (향후)
  reminderEnabled: boolean;
  reminderInterval: 'weekly' | 'biweekly' | 'monthly';
}
```

### 5.4 ComparisonSession

```typescript
interface ComparisonSession {
  id: string;
  createdAt: string;

  baseAnalysisId: string;    // 기준 분석
  compareAnalysisId: string; // 비교 분석

  // 사용자가 체크한 항목
  actionItems: ActionItem[];
}

interface ActionItem {
  suggestion: string;     // 과거 LLM 제안
  status: 'done' | 'pending' | 'skipped';
  note?: string;
}
```

## 9. 템플릿 예시

### 9.1 위험도 분석

```
위 포트폴리오 이미지를 보고 다음을 분석해주세요:

1. **섹터 분석**
   - 각 종목의 산업 섹터 분류
   - 섹터별 투자 비중 계산
   - 특정 섹터 집중도 (30% 이상이면 경고)

2. **위험 요소**
   - 변동성이 높은 종목 식별
   - 최근 3개월 내 큰 이슈가 있었던 종목
   - 상관관계가 높은 종목들 (분산투자 미흡)

3. **개선 제안**
   - 분산투자를 위해 추가하면 좋을 섹터
   - 비중 조절이 필요한 종목
   - 전체적인 포트폴리오 위험도 평가 (1-10점)

**출력 형식**:
- 섹터 분석은 표로 정리
- 위험 요소는 중요도 순으로 번호 매기기
- 개선 제안은 구체적인 액션 아이템으로 작성
```

### 9.2 리밸런싱 제안

```
위 포트폴리오 이미지를 분석해서 리밸런싱 제안을 해주세요:

1. **현재 상태 분석**
   - 종목별 현재 비중 (%)
   - 시가총액 기준 대형주/중형주/소형주 분류
   - 성장주/가치주 비율

2. **목표 포트폴리오 제안**
   - 균형잡힌 섹터 배분 (권장 비중)
   - 위험 수준별 자산 배분 제안

3. **구체적 액션**
   - 비중 축소할 종목 및 목표 비중
   - 비중 확대할 종목 및 목표 비중
   - 신규 편입 고려 종목 (있다면)
   - 예상 거래 금액

**출력 형식**:
| 종목명 | 현재 비중 | 목표 비중 | 액션 | 거래금액 |
|--------|----------|----------|------|---------|

추가로 리밸런싱 이유를 3줄 이내로 요약해주세요.
```

### 9.3 종목별 체크리스트

```
위 포트폴리오의 각 종목에 대해 다음 체크리스트를 작성해주세요:

각 종목별로:

1. **최근 이슈** (1개월 내)
   - 주요 공시 사항
   - 언론에 보도된 이슈
   - 주가 영향을 줄 만한 뉴스

2. **확인 필요 사항**
   - 다음 실적 발표일
   - 배당 기준일/지급일
   - 주요 이벤트 (IR, 신제품 출시 등)

3. **체크 포인트**
   - 최근 증권사 리포트 변화 (컨센서스 상향/하향)
   - 기관/외국인 수급 변화
   - 기술적 지표 (52주 최고가 대비 현재가)

**출력 형식**: 종목별로 구분해서 정리, 중요도 높은 항목은 ⚠️ 표시
```

## 13. FAQ

**Q: 왜 직접 LLM API를 호출하지 않나요?**
A: 비용을 사용자에게 전가하지 않기 위함입니다. 사용자가 이미 갖고 있는 무료 LLM 계정을 활용합니다.

**Q: LLM 응답을 복붙하는 게 불편하지 않나요?**
A: 이 앱의 핵심 가치는 "분석"이 아니라 "기록과 비교"입니다. 한 번의 복붙으로 영구적인 투자 기록이 남습니다.

**Q: 데이터는 어디에 저장되나요?**
A: 모든 데이터는 사용자 디바이스에만 저장됩니다. 서버로 전송되지 않아 프라이버시가 보장됩니다.

**Q: 다른 기기에서 데이터를 보려면?**
A: JSON 내보내기/가져오기 기능을 제공합니다. 향후 선택적 클라우드 동기화도 고려 중입니다.
